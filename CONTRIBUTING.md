# 贡献指南

感谢你考虑为 Vim IDE Enhanced 做出贡献！

## 🤝 如何贡献

### 报告 Bug

如果你发现了 bug，请创建一个 Issue，包含以下信息：

1. **描述问题** - 清楚地描述发生了什么
2. **重现步骤** - 如何一步步触发这个问题
3. **预期行为** - 你期望发生什么
4. **环境信息**：
   - OS 版本
   - Vim 版本 (`vim --version`)
   - 终端类型
5. **截图** - 如果可能，提供截图

### 提出功能建议

我们欢迎新功能建议！创建 Issue 时请说明：

1. **功能描述** - 你想要什么功能
2. **使用场景** - 为什么需要这个功能
3. **期望行为** - 这个功能应该如何工作

### 提交代码

1. **Fork 仓库**

2. **创建分支**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **进行修改**
   - 遵循现有代码风格
   - 添加必要的注释
   - 确保配置可以正常工作

4. **测试你的修改**
   ```bash
   # 在干净的环境中测试
   mv ~/.vimrc ~/.vimrc.bak
   cp vimrc ~/.vimrc
   vim +PlugInstall +qall
   vim test_file.c  # 测试功能
   ```

5. **提交修改**
   ```bash
   git add .
   git commit -m "Add: 简短描述你的修改"
   ```

   提交消息格式：
   - `Add: 新功能`
   - `Fix: 修复问题`
   - `Update: 更新现有功能`
   - `Doc: 文档修改`
   - `Refactor: 代码重构`

6. **推送到 GitHub**
   ```bash
   git push origin feature/your-feature-name
   ```

7. **创建 Pull Request**
   - 清楚地描述你的修改
   - 说明为什么需要这个修改
   - 如果相关，链接到相关的 Issue

## 📝 代码规范

### Vim Script 风格

1. **缩进** - 使用 2 个空格
2. **注释** - 使用 `"` 开头，中文或英文均可
3. **命名** - 使用描述性的变量名
4. **分组** - 相关配置放在一起，用注释分隔

示例：
```vim
"""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""
" => 插件配置
"""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""
let g:plugin_option = 1       " 选项说明
let g:plugin_enable = 1       " 启用插件
```

### Shell Script 风格

1. **Shebang** - 始终使用 `#!/bin/bash`
2. **错误处理** - 检查命令是否成功
3. **注释** - 说明每个主要步骤
4. **输出** - 提供清晰的用户反馈

## 🔍 测试清单

在提交 PR 前，请确保：

- [ ] 配置在 Vim 8.0+ 版本可以正常工作
- [ ] 所有插件可以正常安装
- [ ] 快捷键没有冲突
- [ ] 文档已更新（如果需要）
- [ ] 没有引入明显的性能问题
- [ ] 在不同终端测试过（如果涉及显示）

## 📚 文档修改

文档同样重要！如果你发现：

- 文档中的错误
- 不清楚的说明
- 缺失的信息
- 可以改进的地方

请随时提交 PR 或 Issue！

### 文档文件

- `README.md` - 英文主文档
- `README_CN.md` - 中文主文档
- `CHEATSHEET.md` - 快捷键速查表
- `CONTRIBUTING.md` - 本文件

## 🎨 添加新插件

如果你想添加新插件：

1. **确保插件质量**
   - 活跃维护
   - 足够流行
   - 功能有用
   - 不与现有插件冲突

2. **添加到 vimrc**
   ```vim
   Plug 'author/plugin-name'
   ```

3. **添加配置**
   - 在相应位置添加插件配置
   - 添加注释说明
   - 如有快捷键，添加到快捷键部分

4. **更新文档**
   - 在 README.md 中添加插件说明
   - 在 CHEATSHEET.md 中添加快捷键（如有）
   - 更新插件列表

## ⚠️ 注意事项

### 不要做的事

- ❌ 不要添加太多依赖
- ❌ 不要添加需要编译的插件（如 YCM）
- ❌ 不要修改用户的其他配置文件
- ❌ 不要添加特定于某个项目的配置
- ❌ 不要使用硬编码的路径

### 应该做的事

- ✅ 保持配置简洁
- ✅ 添加清晰的注释
- ✅ 考虑不同操作系统
- ✅ 提供合理的默认值
- ✅ 文档与代码保持同步

## 🐛 调试技巧

### 查看 Vim 错误

```vim
:messages        " 查看所有消息
:verbose set option?  " 查看选项是在哪里设置的
```

### 测试插件

```bash
# 最小化配置测试
vim --noplugin -u ~/.vimrc
```

### 性能分析

```vim
:profile start profile.log
:profile func *
:profile file *
" 执行一些操作
:profile pause
:qa
```
